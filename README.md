# Software Testing Methodologies

This repository contains **hands-on software testing work** completed as part of the **Software Testing Methodologies** course in the **Software Quality Assurance & Test Engineering** program at **Conestoga College**.

The work demonstrates practical application of **core software testing methodologies**, including manual testing, unit testing, integration testing, system testing, acceptance testing, performance testing, and database testing.

---

## What This Repository Includes

### Manual & Functional Testing
- Designed **manual test cases** for functional validation
- Covered valid, invalid, boundary, and negative test scenarios
- Applied testing logic to real programs such as **triangle classification**
- Documented expected vs actual results and test status

### Test Planning & Test Case Management
- Created **test plans and test scenarios**
- Managed test execution workflow using **Kanban-style boards**
- Tracked test progress through **To Do / In Progress / Done** stages
- Practiced requirement analysis and test coverage planning

### Unit & Integration Testing
- Applied **unit testing techniques** such as Boundary Value Analysis
- Implemented **integration testing** using Python and PyTest
- Executed tests through GitHub repositories and verified results
- Practiced version control and test execution in realistic environments

### System & Acceptance Testing
- Created **Acceptance Test Plans**
- Defined **Acceptance Criteria (ACC)** documents
- Prepared **Acceptance Test Status Reports**
- Evaluated system readiness based on pass/fail results and defects

### Performance Testing
- Conducted **performance and load testing** using **Apache JMeter**
- Configured thread groups, samplers, and listeners
- Analyzed summary reports to evaluate system performance under load

### Database & Mutation Testing
- Performed **database testing using SQL**
- Validated insert, update, delete, and data integrity operations
- Executed **mutation testing** to verify test effectiveness against faulty data
